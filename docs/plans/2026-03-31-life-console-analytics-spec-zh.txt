Life Console 埋点与 Dashboard 规格
日期：2026-03-31

1. 文档目标

这份文档把已经确认的产品指标，进一步落到实现层，回答 4 个问题：
1. App 需要埋哪些点
2. 每个埋点要带哪些字段
3. 各项指标如何从事件计算出来
4. 第一版 Dashboard 应该长什么样

2. 埋点原则

1. 只记录能帮助提升执行和清晰度的数据
2. 尽量用事件自动计算，而不是依赖人工报表
3. 区分客观行为数据和主观反思数据
4. 命名稳定，避免 App 端和 Dashboard 端口径漂移
5. 字段要足够支撑后续 AI 诊断

3. 事件分类

A. 会话与导航
- app_opened：用户打开 App
- screen_viewed：用户进入某个主要页面
- dashboard_loaded：Today 首页完成加载

B. 计划与关键项确认
- key_habit_confirmed：确认今日关键习惯
- key_mission_declared：声明今日关键任务
- mission_output_type_selected：选择今日输出物类型
- habit_weight_updated：手动标记某习惯为当前阶段关键

C. 提醒与执行
- reminder_sent：系统发出提醒
- reminder_opened：用户点开提醒
- task_started：用户开始执行
- task_completed：用户标准完成
- task_minimum_completed：用户最低完成
- task_makeup_completed：用户补做完成
- task_missed：任务最终失败

D. 产出与里程碑
- visible_output_logged：记录可见产出
- visible_output_validated：确认该产出有效
- milestone_created：创建里程碑
- milestone_progressed：推进里程碑
- milestone_completed：完成里程碑

E. 日记与复盘
- journal_started：开始 App 内日记
- journal_submitted：提交 App 内日记
- paper_journal_reminder_sent：纸质日记提醒
- paper_journal_confirmed：确认纸质日记已完成
- main_line_check_answered：回答主线跑偏检查
- control_score_self_rated：提交掌控感周评分

F. AI 教练
- ai_setup_suggestion_generated：AI 生成设置建议
- ai_diagnosis_generated：AI 对失败做诊断
- ai_recommendation_accepted：用户接受 AI 建议
- ai_recommendation_dismissed：用户忽略 / 拒绝 AI 建议

6. 指标与埋点映射

结果指标
- 每周可见产出天数
  - 事件来源：visible_output_validated
  - 计算方式：统计一周内至少有 1 次有效可见产出的自然日数量

- 周里程碑推进率
  - 事件来源：milestone_progressed、milestone_completed
  - 计算方式：有里程碑推进的周数 / 总观察周数

- 关键任务声明后完成率
  - 事件来源：key_mission_declared、task_completed
  - 计算方式：已完成声明任务数 / 已声明任务总数

- 关键基础习惯综合稳定度
  - 事件来源：task_completed、task_minimum_completed、task_makeup_completed、task_missed、habit_weight_updated
  - 计算方式：按完成状态与动态权重计算周得分

- 晚间收尾稳定度
  - 事件来源：task_started、task_completed（针对睡前收尾任务）
  - 计算方式：按时进入睡前流程的天数 / 计划天数

诊断辅助指标
- 掌控感周评分
  - 事件来源：control_score_self_rated
  - 计算方式：周均分 + 4 周滚动均值

- 主线跑偏检查
  - 事件来源：main_line_check_answered
  - 计算方式：回答 yes 的天数 / 有回答的天数

行为指标 / Product Health
- 全链路执行转化率
  - 事件来源：reminder_sent、task_started、task_completed、visible_output_validated
  - 计算方式：开始率、完成率、产出率的加权综合

- 关键项确定速度
  - 事件来源：app_opened、key_habit_confirmed、key_mission_declared
  - 计算方式：从打开 App 到两个关键项都确认的中位秒数

- 主线一致率
  - 事件来源：main_line_check_answered
  - 计算方式：回答未跑偏的天数 / 有回答的天数

8. Dashboard 结构

A. 顶部总览
目的：一眼回答“我这周是否更有掌控感了”
- Visible Output Days
- Foundation Stability
- Funnel Score
- Main-line Alignment

B. 结果区
模块 1：项目推进
- 7 天产出条形图
- 产出物列表
- 里程碑进度卡
- 关键任务完成率

模块 2：生活节律稳定
- 基础习惯稳定度主卡
- 权重拆解
- 晚间收尾趋势
- 纸质日记达成率

C. 产品健康区
模块 3：执行转化
- Reminder -> Start -> Complete -> Output 漏斗
- Hard vs Soft 对比
- 不同提醒类型对比

模块 4：决策清晰度
- 关键项确定速度趋势
- 主线一致率热力图
- 跑偏原因分布

D. 诊断区
模块 5：主观感受 vs 客观现实
- 掌控感周评分 vs 基础稳定度
- 掌控感周评分 vs 可见产出天数
- 主观和客观偏差最大的周列表

11. 实现待确认问题

1. V1 的可见产出是否必须上传证据
2. task_started 是纯手动、纯推断，还是两者结合
3. 如果用户不主动点开始收尾，系统如何识别晚间收尾
4. 纸质日记后续是否允许拍照证明

12. 建议

V1 先围绕下面 4 个东西搭建即可：
- 一套稳定事件命名
- 一张 daily_task_fact
- 一张 weekly_user_summary
- 一个 5 模块 Dashboard

这已经足够同时支撑产品优化和 AI 教练，不需要一开始把数据系统做得过重。
